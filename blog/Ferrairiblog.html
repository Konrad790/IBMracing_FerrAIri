2. Research

The first part of our project was research. First we completed a lot of IBM skillsbuild courses on AI and Machine Learning Algorithms. Then we started thinking of possible approaches. Our main idea was of course reinforcement learning, but in order to increase our chances we decided to develop a rule-based model at the same time. While discussing our plan with IBM Bob we came across a technique called behavioral cloning, which we later used to be able to fine tune the model created with the expert data.

So finally we split into teams, where we were working on training rule-based models and rl models separately. From that moment we were spending most of the time experimenting with the reward in order to maximize our model’s performance.

3.Training the driver

Rule-Based model:

        It’s a model that drives based on provided parameters, like target speed or center-line correction.

        To train it we run the given process multiple times:

  • mutate selected parameters
  • run TORCS
  • compare the result: keep the candidate only when the best known lap was improved

        As a result it achieved a great lap time of 101.548 seconds

Behavioral Cloning:

        We decided to collect data from the rule-based model and then train a supervised learning model with it. Because of this we had a great rl model from the start!

Reinforcement Learning:

        After behavioral cloning we started training the rl model using SAC. We still had some problems      with adjusting both observation space and reward, but after multiple tries we finally created a driver capable of completing the lap.

3.1 Further Work and Ideas

Future work should focus on training on increasingly complex tracks to improve model generalization. Additionally, we plan to experiment with a quadratic penalty for steering oscillations relative to vehicle speed. This approach would mathematically account for the increase in centripetal acceleration, forcing the agent to prioritize smooth cornering at higher velocities. Further improvements could also involve augmenting the network architecture with LSTM layers for better trajectory planning.

5.Problems and how we solved them

Our approach relied on iteratively balancing the reward function to eliminate pathological agent behaviors. By observing and systematically addressing these issues, such as deliberately going off the track to minimize the penalty in given situations or unwillingness to explore faster routes, we evolved the agent from chaotic movement and risk aversion to a more smooth and stable performance.
